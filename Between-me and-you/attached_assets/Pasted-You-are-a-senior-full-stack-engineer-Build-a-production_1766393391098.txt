You are a senior full-stack engineer. Build a production-ready MVP web app called "بيني وبينك".

0) Goal

A calm Arabic (Saudi white) web app where the user vents freely, then the AI asks one question per turn, and eventually reveals:

“مشكلتك مو … مشكلتك …”

A short explanation based on the user’s words

One small actionable step (not a life plan)

The app must be simple, clean, RTL by default, and safe (no medical diagnosis).

1) Tech stack

Next.js 14+ (App Router)

TypeScript

No database in MVP

Server-side OpenAI call via an API route

Minimal CSS (plain CSS modules or globals). No heavy UI frameworks.

3) UX rules (must implement)

RTL layout by default (dir="rtl")

Show a “typing…” indicator while waiting for server response

4) Backend API route

Create POST /api/chat (Next.js route handler):

Request body: { messages: {role, content}[] }

It will call OpenAI Responses API server-side

Reads key from process.env.OPENAI_API_KEY

Never expose key client-side

Return JSON: { reply: string }

Handle errors gracefully with a friendly Arabic error message

5) OpenAI call

Use OpenAI Responses API via fetch("https://api.openai.com/v1/responses", ...).

Put the “SYSTEM/DEVELOPER PROMPT” content as the top instruction input

Then include the conversation messages

Choose a reasonable model default (make it configurable via env OPENAI_MODEL, fallback to a commonly available one)

Keep temperature moderate (e.g. 0.6) to allow natural tone without hallucinating too much

Ensure the assistant replies in Arabic Saudi white tone

6) Safety constraints (must be in prompt + code)

Encourage seeking professional help or emergency services

Not continue “analysis mode”
(Enforce through system prompt)
